# 主运行流程 agent_loop

`application/agent_loop.py` 是整个智能体的发动机。

它的职责不是“回答问题”，而是组织一轮又一轮的工作：维护上下文、调用模型、执行工具、处理后台结果、触发压缩，直到模型给出最终回复。

## 从用户输入到模型工作

入口在 `application/repl.py`：

1. 用户在命令行输入问题。
2. 程序把问题包装成 `HumanMessage`。
3. 把这条消息加入 `history`。
4. 调用 `agent_loop(history, services, tool_runtime, system_prompt)`。
5. 主循环开始工作。

这里的 `history` 就是模型每轮能看到的对话上下文。

## 主循环做了什么

每一轮循环大致做这些事：

1. 给消息补 UUID。
2. 尝试调度后台记忆提取。
3. 裁剪过长工具结果。
4. 对较早工具结果做轻量摘要。
5. 估算当前上下文 token。
6. 如果上下文太长，触发 session memory 压缩。
7. 如果还是太长，触发 context collapse。
8. 如果超过硬上限，触发 legacy compact。
9. 收集后台任务结果。
10. 读取团队收件箱。
11. 调用大模型。
12. 如果模型不调用工具，结束本轮并返回。
13. 如果模型调用工具，逐个执行工具，把结果放回上下文。

这就是智能体区别于普通聊天机器人的地方：模型不是一次性输出答案，而是在“思考、行动、观察、再思考”的循环里完成任务。

## 为什么要给消息加 UUID

`session_memory.py` 中的 `ensure_message_uuid` 会给每条消息贴一个稳定 ID。

这样做是为了让系统知道：

- 哪些消息已经被长期记忆覆盖了。
- 哪些消息还没有被总结。
- 压缩时应该从哪里切开。
- 压缩后哪些最近消息必须原样保留。

没有 UUID，系统只能靠列表位置判断边界。列表一旦被压缩、插入、重排，就很容易出错。

## 为什么每轮都要做轻量压缩

工具结果可能非常长。例如搜索结果、文件内容、命令日志都可能一下子占掉大量上下文。

主循环每轮都会调用：

- `apply_tool_result_budget`
- `microcompact`

它们的作用是：

- 单条工具结果太长时，只保留头尾。
- 旧的工具结果改成摘要。
- 最近的工具结果仍保留原文，方便继续调试。

这是一种非常实用的策略：旧证据保留概要，当前证据保留细节。

## 模型和工具的配合方式

模型调用后会返回一条 `AIMessage`。

如果这条消息没有 `tool_calls`，说明模型认为任务可以直接回答。主循环结束。

如果有 `tool_calls`，主循环会：

1. 取出工具名。
2. 取出工具参数。
3. 在 `tool_runtime.handlers` 里找到对应执行函数。
4. 执行工具。
5. 把结果包装成 `ToolMessage`。
6. 继续下一轮循环。

所以模型只负责“决定用什么工具和参数”，真正的文件读写、命令执行由项目代码负责。

## 工具缓存设计

主循环里有一个小型工具缓存：

- 可缓存工具：`glob_files`、`grep_content`、`read_file`、`read_file_segment`。
- 会使缓存失效的工具：`write_file`、`edit_file`。

设计原因很简单：

- 读文件、搜索文件经常重复，可以缓存，减少重复开销。
- 一旦写入或编辑文件，之前的读取结果可能过期，必须清空缓存。

这是智能体项目里很常见的工程优化：读操作可以缓存，写操作要清缓存。

## Todo 提醒机制

主循环会检查模型是否使用了 `TodoWrite`。

如果 Todo 里还有未完成事项，但模型连续几轮没有更新 Todo，系统会插入一条提醒：

```text
<reminder>Update your todos.</reminder>
```

这个机制的目的不是限制模型，而是防止模型在复杂任务中忘记自己列过的计划。

## 后台任务和团队消息

主循环还会不断检查：

- 后台命令是否完成。
- 队友是否发来消息。

如果有结果，它们会被包装成新的 `HumanMessage` 放回上下文。

这说明该项目把“外部事件”也当成上下文的一部分。智能体不是只处理用户输入，也会处理后台执行结果和协作者消息。

## 主循环的核心设计思想

`agent_loop` 的核心思想可以总结为：

1. 模型负责决策，代码负责执行。
2. 每轮执行前先维护上下文健康。
3. 长任务要持续沉淀记忆。
4. 工具调用结果必须回流到上下文。
5. 最终回答只是循环停止时的自然结果。
